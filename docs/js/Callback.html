Callback class

Converts a local Dart function to a callback that can be passed to JavaScript.

A callback can either be:

  • single-fire, in which case it is automatically invalidated after the first invocation, or

  • multi-fire, in which case it must be explicitly disposed.

Implements

Serializable<FunctionProxy>

Constructors

new Callback.many(Function f, {bool withThis: false}) #

Creates a multi-fire Callback that invokes f. The callback must be explicitly disposed to avoid memory leaks.

new Callback.once(Function f, {bool withThis: false}) #

Creates a single-fire Callback that invokes f. The callback is automatically disposed after the first invocation.

Methods

dynamic dispose() #

Disposes this Callback so that it may be collected. Once a Callback is disposed, it is an error to invoke it from JavaScript.
